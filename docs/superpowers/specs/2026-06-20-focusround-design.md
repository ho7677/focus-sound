# FocusRound — 설계 스펙

**날짜:** 2026-06-20  
**상태:** 승인됨

---

## 1. 개요

복싱 라운드 구조를 가진 웹캠 기반 집중력 측정 타이머 앱.  
화면을 보지 않아도 소리만으로 집중/휴식 상태를 구분할 수 있으며, 세션 종료 후 실제 집중 시간 리포트를 제공한다.

---

## 2. 사용자 시나리오

1. 앱을 열고 **라운드 수 / 집중 시간 / 휴식 시간**을 설정한다.
2. 시작 버튼을 누르면 웹캠 권한을 요청하고 첫 라운드가 시작된다.
3. 사용자는 다른 화면(개발, 독서 등)으로 전환해 작업한다.
4. 소리만으로 집중/휴식 전환을 인식한다.
5. 모든 라운드 완료 후 집중률 리포트를 확인한다.

---

## 3. 화면 구성 (3개 뷰)

### 3-1. 세팅 화면
- 라운드 수 입력 (기본: 4)
- 집중 시간 입력 (기본: 25분)
- 휴식 시간 입력 (기본: 5분)
- 시작 버튼

### 3-2. 타이머 화면
- 현재 라운드 / 전체 라운드 표시 (예: Round 2 / 4)
- 현재 상태 표시: 집중 / 휴식
- 남은 시간 (MM:SS, 크게 표시)
- 비집중 누계 시간 (작게 표시)
- 웹캠 소형 프리뷰 (얼굴 감지 상태 표시)
- 일시정지 / 종료 버튼

### 3-3. 리포트 화면
- 총 세션 시간
- 실제 집중 시간 (녹색)
- 비집중 누계 시간 (빨간색)
- 집중률 % 표시
- 비집중 감지 횟수
- 다시 시작 버튼

---

## 4. 소리 규칙

| 이벤트 | 소리 |
|--------|------|
| 집중 라운드 시작 | 딩동댕 (3타, 높은 음) |
| 휴식 시작 | 딩 (1타, 낮은 음) |
| 비집중 감지 (1분 무움직임) | 삐 경보 (짧고 날카롭게) |
| 모든 라운드 완료 | 딩동댕~♪ (완료 멜로디) |

모든 소리는 Web Audio API로 생성 (외부 파일 불필요).

---

## 5. 웹캠 감지 로직

**라이브러리:** MediaPipe FaceMesh (CDN 로드)

### 감지 규칙
- **움직임 측정:** 이전 프레임 대비 랜드마크 좌표 변화량 합산
- **비집중 판정:** 변화량이 임계값 미만으로 **60초 연속** 유지되면 비집중 누적 시작
- **자리이탈 판정:** 얼굴이 **10초 이상** 감지되지 않으면 자리이탈로 분류
- 비집중(멍함) / 자리이탈 모두 "비집중 시간"으로 합산
- 비집중 감지 시 삐 경보음 1회 발생
- 다시 움직임 감지되면 집중 상태로 복귀

### 웹캠 프리뷰
- 타이머 화면 우측 하단에 소형(130×100px) 표시
- 얼굴 감지 상태: 🟢 감지됨 / 🔴 미감지 텍스트 표시

## 6. 데이터 모델

```js
// 세션 설정
{
  rounds: Number,        // 라운드 수
  focusMinutes: Number,  // 집중 시간(분)
  breakMinutes: Number,  // 휴식 시간(분)
}

// 세션 결과 (LocalStorage 저장)
{
  date: String,          // ISO 날짜
  totalSeconds: Number,  // 총 세션 시간(초)
  focusSeconds: Number,  // 집중 시간(초)
  unfocusSeconds: Number,// 비집중 시간(초)
  unfocusCount: Number,  // 비집중 감지 횟수
  focusRate: Number,     // 집중률 (0~100)
}
```

## 7. 기술 스택

- **언어:** 순수 HTML / CSS / JavaScript (프레임워크 없음)
- **AI 감지:** MediaPipe FaceMesh (`@mediapipe/face_mesh` CDN)
- **소리:** Web Audio API (OscillatorNode로 직접 생성)
- **저장:** LocalStorage (세션 기록)
- **빌드:** 기존 `npm run build` (scripts/build-static-site.mjs)
